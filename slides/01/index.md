# 「アプリ」ってなんだろう？
- ハードウェア
  - マウス, キーボード, ディスプレイ, etc.
- ソフトウェア
  - システム・ソフトウェア
    - macOS, Windows, Linux
    - iOS, Android
  - **アプリケーション・ソフトウェア**
    - *デスクトップ・アプリケーション*
      - ウェブブラウザ (Safari, Firefox, Google Chrome, etc.)
      - グラフィックスソフト (Illustrator, Photoshop, GIMP, etc.)
      - テキストエディタ (VSCode, Notepad, Zed, etc.)
      - 大半の PC 用ゲーム
    - *コマンドライン・アプリケーション*
      - git, vim, curl, etc.

---

アプリとは**アプリケーション (Application) ・ソフトウェア**のことを指す。
Application には*適応・応用*というような意味がある。
ユーザーはその時々の目的に応じて適切なアプリを選択し、目的の達成のために利用する。

| 目的 | アプリ |
| :-- | :-- |
| 情報収集 | Firefox（ウェブブラウザ） |
| デザイン | Illustrator（グラフィックスソフト） |
| コーディング | VSCode（テキストエディタ） |
| 暇つぶし | Doom（ゲーム） | 

---

# ウェブアプリ
ウェブブラウザ上で動作するアプリのことを**ウェブ・アプリケーション（ウェブアプリ）** と呼ぶ。
一般的なウェブサイトは複数の文書（ドキュメント）の総体として、それらの*閲覧*を主な機能としてユーザーに提供する。

それに対しウェブアプリは何らかの形で*ユーザーの入力*を受け付けるインターフェースを有し、
それらを介して入力された情報に応じ、ブラウザに表示する内容を*動的*に変化させる。

---

## ウェブアプリの利点
- アプリを DL する必要がない。ストレージの容量も節約できる。
- アプリを更新する必要もない。常に最新のバージョンを使用できる。
- OS に関係なく利用できる。（クロスプラットフォーム）
  - これはアプリの開発者にとっての利点でもある。

<hr>

## ウェブアプリの欠点
- ネイティブアプリに比べると動作が遅い。
- オフラインでは使用できない。
- アプリ提供者側の都合で使用不可になったり、使い勝手が変化してしまうリスクがある。
  
---

# ウェブアプリは「何」で出来ている？
どんなウェブアプリもユーザーのブラウザ上で動作しなければならない以上、
一般的なウェブサイトと同様、**HTML**, **CSS**, そして **JavaScript** で作られている。

ただし、中にはブラウザだけでは完結せず、都度*サーバー*とのデータのやりとりを必要とするウェブアプリもある。その場合、サーバー側のプログラムが何の言語で書かれているか、ユーザーが直接知ることはできない。

そのようなウェブアプリにおいては、
- ブラウザ上で動作する部分を「**フロントエンド (Front-end)**」  
- サーバー側で動作する部分を「**バックエンド (Back-end)**」  

と呼んで区別することが多い。

---

当講義においては、基本的にバックエンドは扱わず、
ブラウザ上で完結するアプリの開発に取り組んでいこうと思う。

---

# 開発環境の構築

---

## コマンドライン・アプリケーション
ダブルクリックで実行するデスクトップアプリとは異なり、
コマンドライン・アプリケーションは基本的に**ターミナル**上で使用するものだ。
**CLI アプリ**とも呼ばれる。
CLI とは *Command Line Interface（コマンドライン・インターフェース）* の略で、
GUI (Graphical User Interface) とは対照的な概念といえる。

---

早速ターミナルを開いてみよう。

```sh
%
```

表示されている `%`（または `$` ）記号の後に続けて「**コマンド**」と呼ばれる文字列を入力する。
`Enter` キーをタイプするとコマンドを実行することができる。

<hr>

試しに `ls` というコマンドを実行してみよう。

```sh
% ls
```

すると `Pictures` や `Documents` といったフォルダの一覧が表示されるはずだ。

---

<!-- _class: small -->

### 代表的なコマンド

| コマンド名 | 説明 | 用例 |
| :-- | :-- | :-- |
| `echo`  | 文字列を出力する | `echo "Hello!"`
| `pwd`   | 現在の作業ディレクトリを表示する | `pwd`
| `ls`    | ファイル & フォルダの一覧を表示する | `ls`
| `mkdir` | フォルダを作成する | `mkdir Test`
| `cd`    | ディレクトリを移動する | `cd Desktop/Downloads`
| `cat`   | ファイルの内容を出力する | `cat file.txt`
| `cp`    | ファイルをコピーする | `cp alfa.txt bravo.txt`
| `mv`    | ファイルを移動する | `mv file.txt Desktop/`
| `rm`    | ファイルを削除する | `rm file.txt`

---

ターミナルには**作業ディレクトリ (Working Directory)** という概念が存在する。
ディレクトリは「*📁 フォルダ*」と読み替えても差し支えない。
要するに、**自分が現在どのフォルダに居るのか**を示すものだ。

<hr>

現在の作業ディレクトリは `pwd` コマンドで確認できる。
作業ディレクトリを変更する、つまりフォルダを移動するには `cd` コマンドを使う。

```sh
% cd 移動先ディレクトリ名
```

ターミナルを使用する際は必ず以下の事を心掛けよう。 
- 最初に `cd` コマンドで**作業場となるフォルダまで移動**する。
- 他のコマンドを実行する前に、自分が**正しい作業ディレクトリ**にいるかを確認する。

---

### Mac の大まかなディレクトリ構成

```sh
📁 /              # ルート
├── 📁 System/        # OS のシステムファイル群
├── 📁 Applications/  # アプリケーション群
│   ├── Safari.app
│   ├── Mail.app
│   └── ...
└── 📁 Users/         # 各ユーザーのホームディレクトリ群
    ├── 👤 alice/         # alice のホームディレクトリ
    ├── 👤 bob/           # bob のホームディレクトリ
    └── 👤 charlie/       # charlie のホームディレクトリ
        ├── 📁 Desktop/       # charlie のデスクトップ
        ├── 📁 Downloads/     # charlie のダウンロードファイル群
        └── ...
```

普段デスクトップに表示されているファイルやフォルダは全て、
実際は *`/Users/自分のユーザー名/Desktop`* というディレクトリ下に存在している。

---

### プロジェクトを作成する
ウェブアプリの開発を始めるにあたり、まずは**制作の起点となるフォルダ**を作成しよう。
作成する場所は自分のホームディレクトリ以下であれば基本的にどこでもよい。
今回は分かりやすいように*デスクトップの直下*に作成することとする。

まずは `cd` コマンドで `/Users/自分のユーザー名/Desktop` に移動する。

```sh
% cd
```

このように*移動先を指定せず*に `cd` コマンドを実行することで、
自分の*ホームディレクトリ*、すなわち *`/Users/自分のユーザー名`* に移動することができる。
そのあとは単に、

```sh
% cd Desktop
```

とだけ実行すれば `/Users/自分のユーザー名/Desktop` に移動することができる。

---

ちなみに、ディレクトリ名やファイル名はいちいち全て手力しなくても、
**途中まで入力してから `Tab` キー**を押せば自動で補完される。

<hr>

もし移動先のディレクトリを間違ってしまったら、

```sh
% cd ..
```

で**一つ上の階層**のディレクトリに戻ることができる。

<hr>

また、**`↑`** キーをタイプすると、**過去に実行したコマンドの履歴を遡る**ことができる。覚えておこう。

---

デスクトップに移動できた所で、制作の起点となるフォルダを作成しよう。
フォルダを作成するコマンドは **`mkdir`** だ。*"Make Directory"* と覚えよう。

```sh
% mkdir 新規フォルダ名
```

このように、スペースの後に作成したいフォルダ名を続ける。
フォルダ名については、ひとまず今回は `HelloP5` としてほしい。

```sh
% mkdir HelloP5
```

フォルダが作成できたら `cd` コマンドで中に入ろう。

```sh
% cd HelloP5
```

---

今後、暫くはこの `HelloP5` フォルダをあらゆる作業の起点として使用することとする。

<hr>

今回に限らず、プログラマーが何らかのプロジェクト<small>（特定のウェブサイトやアプリの制作など）</small>
に参加する際は、*各プロジェクトに専用のフォルダ*を割り当て、そのフォルダ内でコードを管理する。
このようなフォルダを概して「**プロジェクト・ルート (Project Root)**」と呼ぶ。

---

#### ここまでのまとめ :: ターミナルの操作
| コマンド or 入力キー | 説明 |
| :-- | :-- |
| `cd` | 自分のホームディレクトリに移動
| `cd 移動先ディレクトリ名` | 任意のディレクトリに移動
| `cd ..` | 一つ上の階層のディレクトリに移動
| `pwd` | 現在地（作業ディレクトリ）の確認
| `ls` | 作業ディレクトリ内のファイル & フォルダ一覧を表示
| `mkdir 新規フォルダ名` | フォルダを作成する
| `Tab` キー | ファイル名の自動補完
| `↑` キー | コマンドの実行履歴を遡る

---

## 開発ツールの導入
ウェブアプリの開発を始めるにあたり、まずは **[Node.js](https://nodejs.org/en)** という CLI アプリが必要になる。
Node.js は *JavaScript の実行環境*の一つだ。

知っての通り、一般的に JS はウェブサイト上で実行されるものだが、Node.js を導入することで、
ウェブブラウザを介することなくターミナル上で *`*.js` ファイルを直接実行*することが可能になる。

<hr>

また、Node.js には **NPM <small>(Node Package Manager)</small>** という CLI アプリも付属している。
NPM を利用することで、*ネット上に公開されている様々な JS プログラム*を自分のマシンにインストールしたり、自分の JS プログラムに*モジュール*として組み込んだりすることが簡単に行える。

---

### Homebrew
Node.js の導入には公式サイトから DL できるインストーラーを利用してもいいのだが、
**[Homebrew](https://brew.sh/)** を利用すれば以下のコマンドだけでインストールが完了する。

```sh
% brew install node
```

Homebrew は macOS<small>（または Linux）</small>用の*パッケージ管理ソフト*の一つで、
任意の CLI アプリを上記のようなコマンド一つでインストールしてくれるユーティリティだ。
まだ未導入であれば、この機会に導入してしまおう。

Homebrew の導入方法は[公式サイト](https://brew.sh/)に書かれている通りで問題ない。

---

### Node.js の動作確認
Node.js のインストールが完了すると、*`node`* コマンドが使用可能になる。
ターミナルで確認してみよう。

```
% node
Welcome to Node.js vXX.XX.XX.
Type ".help" for more information.
>
```

大体このような表示になれば OK。
`vXX.XX.XX` の部分はインストールされた Node.js のバージョンを示している。

`>` の後に続けて JS のコードを入力すると即座に実行され、結果が出力される。
この状態を *REPL* <small>(Read, Eval, Print, Loop)</small> モードと呼ぶ。
REPL モードでは **`cd` や `ls` といった通常のコマンドは受け付けない**ので要注意。

---

試しに `console.log('Hello')` と入力してみよう。

```sh
> console.log('Hello')
Hello
undefined
```

すると、このように出力されるはずだ。
**`console.log()` は引数に渡された値を文字列として出力する関数**だ。
今後何かとお世話になる関数なので覚えておこう。
今回は `'Hello'` という値を渡したので `Hello` と出力されたというわけだ。

その下の行には関数の*返り値*（ `return` される値）が表示されるが、
`console.log()` 関数は返り値を持たないため `undefined` と表示されている。
`undefined` は「*未定義*」という意味だ。

以上を確認できたら *`.exit`* と入力して REPL モードを抜けよう。
